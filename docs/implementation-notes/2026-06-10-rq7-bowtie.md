# 2026-06-10 — RQ-7 Bow-tie analysis visualisation

**Commit:** `<sha>` feat(risk): bow-tie analysis projection (RQ-7)

ISO 31010 bow-tie: threat → risk-event → consequence with control barriers on
each side. The quickest win — a **pure read projection** of data IC already has
(no new schema).

## Design

- **`bowtie-projection.ts`** — `buildBowTie(risk, controls)` is **pure**:
  splits `Risk.threat` narrative into threat sources (carrying RQ-1 TEF/vuln),
  classifies linked controls into preventive barriers (PREVENTIVE/DETERRENT, left)
  vs mitigating barriers (DETECTIVE/CORRECTIVE/COMPENSATING, right), and
  decomposes the FAIR PLM (productivity/response/replacement/secondary) into
  separate consequence nodes — falling back to a single `resolveALE` loss node.
  `toXyFlowGraph` lays it out (threats left → event centre → consequences right,
  barriers between) for a future ReactFlow canvas. `projectBowTie` is the DB
  wrapper (Risk + RiskControl + Control in two reads).
- **Route** — `risks/[id]/bowtie` (GET projection + xyflow graph).
- **UI** — a **Bow-Tie tab** on the risk detail page rendering the classic
  five-column layout (threats · preventive · event · mitigating · consequences),
  barrier colour = control effectiveness.

## Decisions

- **No stored graph** — computed at read time from existing relations; zero
  migration, always in sync with the risk + its controls.
- **Five-column card layout** ships now (functional + token-clean); the
  `toXyFlowGraph` output is ready for an interactive ReactFlow upgrade later.
