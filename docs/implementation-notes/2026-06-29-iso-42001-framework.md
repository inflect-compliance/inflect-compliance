# 2026-06-29 — ISO/IEC 42001:2023 as an assessable framework

**Commit:** _(this PR)_ `feat(frameworks): ISO/IEC 42001 AI management system`

## Context

First of a three-PR AI-governance sequence (ISO 42001 → EU AI Act → three-way
crosswalk). It adds the "ISO 27001 for AI" — the AI **management system**
standard — so AISVS technical verification (already shipped, #1328) can later
roll up into a management system and EU AI Act obligations. This PR is the
ISO 42001 framework only; the crosswalks come in the final PR.

## Design

ISO 42001 ships as framework **content** on IC's data-driven library machinery
(no new code paths), mirroring ISO 27001 / AISVS. Structure:
- **Management clauses 4-10** (Harmonized Structure, AI-specialised) → depth-1
  grouping nodes with 24 assessable sub-clause requirements (4.1–10.2).
- **Annex A objectives A.2-A.10** → depth-1 groupings with 38 assessable
  controls (A.2.2 … A.10.4).

62 assessable nodes total. Seeds the framework (`ISO42001`/`2023`,
`ISO_STANDARD`) + requirements + one control template per top-level grouping +
the `ISO42001_BASELINE` pack, all via the generic flow.

## License — ISO copyright (the load-bearing constraint)

ISO 42001 is a **copyrighted ISO standard** — the text CANNOT be redistributed.
IC stores a **structural outline**: clause + Annex A control **numbers** with
**short paraphrased titles** only — never verbatim ISO prose — exactly how
`iso27001-2022.yaml` already handles ISO copyright. Every requirement
description carries a `(Paraphrase)` marker; the `copyright` field disclaims
reproduction and points at iso.org. The ratchet enforces a length ceiling per
title + the paraphrase marker so a verbatim paste fails CI.

This differs from the other two frameworks in the sequence: AISVS is
CC-BY-SA-4.0 (index-not-prose for ShareAlike reasons); the EU AI Act is public
domain (its article text may be used). The per-source license matrix is
documented in full in the final bundle note.

## Decisions

- **`kind: ISO_STANDARD`** — same as ISO 27001; no migration.
- **No scores_definition** — matches ISO 27001's binary/compliance model
  (the library schema makes scoring optional).
- **Generic install** — control templates grouped per top-level node (clauses +
  Annex objectives); the framework requirements remain the assessable unit, so
  the existing coverage/maturity flows assess ISO 42001 like any framework.
- **Crosswalks deferred** — AISVS→42001 and 42001→EU-AI-Act land in the final
  PR once the EU AI Act framework exists, keeping each PR self-contained.
