# 2026-06-13 — RQ3-OB-E — Respect every user (a11y)

**Commit:** `pending` (branch `claude/rq3-ob-e-a11y`)

## Design

Three loops the visual UI closed but the assistive-tech UI hadn't.

### 1. Warnings that announce themselves

The residual-baseline conflict warning ("Inherent has changed since you opened
the residual draft — your residual now applies to L=4 × I=5. Re-check the
dimensions before saving.") rendered as a plain `<div>` — the user with a
draft open never heard it. Now: `role="status"` + `aria-live="polite"` so a
screen reader announces the warning on mount without stealing focus.

The FAIR calibration warnings (the "your PERT triple looks degenerate" line on
the quantification panel) already routed through `<InlineNotice>`, which bakes
`role="status"` + `aria-live="polite"` into the primitive. The ratchet locks
the continued use of the primitive so a future "raw `<div>`" regression is
caught.

### 2. The grid keeps its promise

`<RiskMatrix>` advertises `role="grid"` to AT — the WAI-ARIA spec promises
arrow-key navigation in return. The matrix was Tab-only. Now: **roving
tabindex** (exactly one cell tabbable at a time) + **arrow keys** to steer +
**Home / End** to jump to the row's edges. Edges **clamp** (not wrap) because
the matrix is geographic — wrapping likelihood→impact would confuse the
mental model.

The state lives in the matrix; the cell is direction-agnostic:

```text
RiskMatrix  ──── focusedKey ─────► RiskMatrixCell.tabbable
              ◄── onArrowKey ─── (Arrow/Home/End)
```

Focus moves imperatively after React commits (`requestAnimationFrame` →
`querySelector('[data-testid=risk-matrix-cell-${l}-${i}]')`), so tabIndex
updates and DOM focus land in the same paint.

Roving tabindex only engages when `onCellClick` is wired. A read-only matrix
has no destination — every cell stays at `tabIndex=-1` (no dead-end stops).
**Enter / Space still fires `onCellClick`** — arrow nav doesn't shadow the
existing keyboard-click contract.

### 3. Motion is a preference

The repo already has a global `@media (prefers-reduced-motion: reduce)` rule
in `src/styles/tokens.css` that collapses `--duration-fast` to 1ms. But that
catches utilities that READ the CSS variable; a literal `transition-colors
duration-150 ease-out` (which `RiskMatrixCell` uses) doesn't. So the cell
gains a per-utility `motion-reduce:transition-none` companion — belt to the
global rule's braces.

The ratchet asserts the guard AND scans the file for any unguarded
`transition-*` utility, so a future PR that adds a new motion class without a
`motion-reduce:` companion fails CI.

## Decisions

- **`role="status"` not `role="alert"`.** The conflict warning IS the kind of
  thing `polite` was designed for — wait for the screen reader to finish its
  current utterance before announcing. `alert` would interrupt the user mid-
  sentence, which the spec reserves for time-critical / error states. This is
  neither.
- **Clamp at edges, no wrap.** Likelihood × impact is geographic, not a list.
  Wrapping from the top-right corner to the bottom-left would lose the mental
  model. Home / End jump within the row only, mirroring the WAI-ARIA grid
  pattern's default.
- **Roving tabindex managed by the matrix, not the cell.** A cell doesn't
  know its neighbours; the matrix does. The cell's `tabbable` prop and
  `onArrowKey` callback are the seam — the cell stays a leaf, the parent
  handles routing.
- **Per-utility `motion-reduce:` guard, not just the global CSS var.** The
  global rule (`@media (prefers-reduced-motion: reduce)` → `--duration-fast:
  1ms`) only catches utilities that READ the var. Tailwind's
  `transition-colors duration-150 ease-out` doesn't. The literal duration
  needed a literal guard.
