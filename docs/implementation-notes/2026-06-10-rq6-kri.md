# 2026-06-10 ‚Äî RQ-6 Key Risk Indicators (KRI)

**Commit:** `<sha>` feat(risk): key risk indicators dashboard (RQ-6)

Leading/lagging indicators with RAG thresholds + reading history ‚Äî the
"canary in the coal mine" early-warning layer (Archer KRI parity).

## Design

- **`computeRag(value, direction, greenMax, amberMax)`** is **pure**: null
  thresholds ‚Üí GREEN; HIGHER_IS_WORSE uses ‚â§ bands, LOWER_IS_WORSE inverts
  (high = good). Unit-tested.
- **Schema** ‚Äî `KeyRiskIndicator` (per-tenant, optional risk link + owner,
  RAG thresholds, direction, frequency, target) + `KriReading` (value + RAG
  computed at write time) + RLS on both + migration.
- **`recordReading`** computes RAG, compares to the previous reading's RAG, and
  flags a **worsening** crossing (GREEN‚ÜíAMBER‚ÜíRED) ‚Äî an improvement never
  breaches. A worsening crossing writes a `KRI_THRESHOLD_BREACH` audit event.
  `batchRecordReadings` (system source) + `getReadings` + `listKris` (attaches
  latest reading + a 12-point sparkline window).
- **Routes** ‚Äî `risks/kri` (list/create), `[kriId]` (patch/delete),
  `[kriId]/readings` (history/record). **UI** ‚Äî `risks/kri` page: RAG cards with
  a unicode sparkline + inline record-reading + create.

## Decisions

- **Breach = worsening crossing only** (rank GREEN<AMBER<RED), so recovering from
  RED‚ÜíGREEN doesn't spam alerts ‚Äî locked by an integration test asserting exactly
  2 breach audit rows across GREEN‚ÜíAMBER‚ÜíRED‚ÜíGREEN.
- **RAG stored on the reading** (write-time) so reads never re-evaluate thresholds.
- Unicode block sparkline (‚ñÅ‚ñÇ‚ñÉ‚ñÑ‚ñÖ‚ñÜ‚ñá‚ñà) on the card ‚Äî no chart primitive / SVG, so
  no extra chart-guard surface; the per-risk detail tab can upgrade to a real
